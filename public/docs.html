API Reference

Complete reference for the CrazyApi REST API. All endpoints return consistent JSON. Use your API key on every request.

Table of Contents
Authentication Base URL Response Format Error Codes Rate Limits Code Examples Endpoint List Self-Hosting

Authentication

Every request requires a valid apikey parameter. You can pass it as a query string or as the X-API-Key header.

GET /api/anime/waifu?apikey=YOUR_KEY # Or via header: GET /api/anime/waifu X-API-Key: YOUR_KEY

Create a free account to get your personal API key. Any non-empty string works in dev mode (when Supabase is not configured).

Base URL

All API calls are made to your deployment URL. For local development:

http://localhost:3000

All endpoints are prefixed with /api/ followed by the category and endpoint name.

Response Format

Every endpoint returns a consistent JSON envelope:

// ✅ Success { "status": true, "creator": "CrazyApi", "result": { ... } // or "count", "query", etc. } // ❌ Error { "status": false, "creator": "CrazyApi", "error": "Human-readable error message" }

The result field can be a string, object, array, or URL depending on the endpoint.

Limits are enforced per API key (daily) and per IP (per minute).

Code Examples

JavaScript (fetch)

const res = await fetch('http://localhost:3000/api/anime/waifu?apikey=YOUR_KEY'); const data = await res.json(); if (data.status) console.log(data.result.url);

JavaScript (axios)

const axios = require('axios'); const { data } = await axios.get('http://localhost:3000/api/search/wiki', { params: { apikey: 'YOUR_KEY', q: 'JavaScript' } }); console.log(data.result.extract);

Python

import requests r = requests.get('http://localhost:3000/api/fun/joke', params={ 'apikey': 'YOUR_KEY', 'category': 'Programming' }) print(r.json()['result'])

cURL

curl "http://localhost:3000/api/tools/ip?apikey=YOUR_KEY&ip=8.8.8.8"

PHP

$response = file_get_contents( 'http://localhost:3000/api/anime/quote?apikey=YOUR_KEY' ); $data = json_decode($response, true); echo $data['result']['quote'];

Self-Hosting

CrazyApi is fully open-source and self-hostable. You own your data.

# 1. Clone & install git clone https://github.com/yourname/crazyapi cd crazyapi npm install # 2. Configure environment cp .env.example .env # Edit .env with your Supabase credentials # 3. Setup database # Paste supabase_schema.sql into your Supabase SQL Editor # 4. Start npm start # production npm run dev # development (nodemon)

Works without Supabase in dev mode — any non-empty apikey is accepted. Contact us for help deploying.
